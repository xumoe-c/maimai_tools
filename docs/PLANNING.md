# 舞萌DX 实用工具站 - 后续规划文档 (Roadmap)

本文档记录了项目的后续开发计划、待办事项及扩展建议。已完成的功能模块请参考 `README.md`。

## 1. 待开发功能 (To-Do List)

### 1.1 🧮 B50 计算器 (Manual Calculator)

* **状态**: 规划中 (入口已预留)
* **目标**: 允许玩家手动输入成绩或修改现有成绩，模拟计算 B50，用于规划推分路线。
* **功能需求**:
  * **手动添加**: 不依赖查分器数据，手动选择歌曲并输入达成率。
  * **沙箱模式**: 基于现有 B50 数据，修改某首曲子的达成率，实时预览 Rating 变化。
  * **目标设定**: 输入目标 Rating，计算需要将哪些曲子推到多少分。

## 2. 体验优化 (UX Improvements)

### 2.1 📱 移动端适配深化

* **当前状态**: 基础适配已完成。
* **优化方向**:
  * 优化表格在小屏幕下的展示 (如版本对照表)。
  * 优化生成器画布在移动端的交互 (缩放、平移)。
  * 增加手势操作支持 (如侧边栏滑动呼出)。

### 2.2 ⚡ 性能优化

* **图片加载**: 实现更激进的图片懒加载策略，或使用 WebP 格式 (需服务端支持或前端转换)。
* **数据缓存**: 优化 `localStorage` 存储结构，避免数据量过大导致溢出 (考虑使用 IndexedDB)。
* **构建体积**: 分析并拆分 vendor chunk，减少首屏加载时间。

### 2.3 🌐 PWA 支持

* **目标**: 使应用可安装，并支持离线访问基础功能 (如版本表、计算器)。
* **实现**: 配置 `vite-plugin-pwa`，缓存关键静态资源。

## 3. 扩展建议 (Extensions)

### 3.1 社区分享功能

* 虽然是纯前端应用，但可以通过生成 "分享码" (Base64/JSON) 的方式，允许用户分享自己的喜好表配置或 B50 模拟数据给他人导入。

### 3.2 更多生成器模板

* **B50 生成器**: 增加更多风格的模板 (如简约风、赛博朋克风)。
* **喜好表**: 增加更多尺寸 (如 5x5, 4x4) 或特殊形状布局。

### 3.3 国际化 (i18n)

* 支持英文/日文界面，服务更多海外玩家。

## 4. 代码维护 (Refactoring)

* **组件拆分**: `ReviewView.vue` 和 `GeneratorView.vue` 代码量较大，考虑进一步拆分业务逻辑到 Composition Functions (Hooks)。
* **类型检查**: 引入 TypeScript (逐步迁移) 以提高代码健壮性。
* **单元测试**: 为核心计算逻辑 (Rating计算、B50排序) 添加单元测试。
